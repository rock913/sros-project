**Date:** 2026-01-21

## Objective
Migrate the `Arxiv` search capability to the MPA/MCP architecture.
Address the technical debt in `graph.py` where Arxiv results are parsed ad-hoc.

## Architecture

### 1. Domain Layer (`agent.domain`)
- **Schema**: Reuse `Paper` from `agent.domain.schemas.paper`.
- **Protocol**: Create `PaperSearcher` in `agent.domain.ports.paper_searcher.py`.
    - Method: `search(query: str, max_results: int) -> List[Paper]`

### 2. Infrastructure Layer (`agent.infrastructure`)
- **Adapter**: `ArxivAdapter` in `agent.infrastructure.tools.arxiv_adapter.py`.
    - Implementation: Use `arxiv` python package directly (cleaner than LangChain wrapper for structured data).
    - Responsibility: Convert raw `arxiv.Result` objects into our domain `Paper` objects.

### 3. MCP Layer (`agent.infrastructure.mcp`)
- **Tool**: `get_arxiv_search_mcp_tool()` in `agent.infrastructure.mcp.tools.arxiv.py`.
    - Handler: Wires inputs to `ArxivAdapter.search`.

## Execution Steps

2.  **Implement Adapter**: Create `ArxivAdapter` (with tests).
3.  **Create MCP Wrapper**: Create `McpTool` factory.
4.  **Register**: Update `entrypoint.py`.
