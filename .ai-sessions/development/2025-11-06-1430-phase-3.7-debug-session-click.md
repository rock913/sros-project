### 🎉 问题已定位！

根据用户提供的调试日志，**问题根因已经100%确认**：

```javascript
[showSessionAnalytics] Called with sessionId: {

  label: 'multi-model fundation model survey',

} type: object  // ← 整个TreeItem对象被传入了！
```

### 根本原因分析

**问题**:
- `showSessionAnalytics` 函数接收到的不是 sessionId 字符串
- 而是**整个 `SessionTreeItem` 对象**

**为什么会这样**:
1. 在 `package.json` 中定义了右键菜单命令：
   ```json
   {
     "command": "auto-researcher.showSessionAnalytics",
     "when": "view == auto-researcher.researchSessionsTree && viewItem == session",
     "group": "2_info@1"
   }
   ```

2. 当用户**右键点击 TreeView 中的 session 节点**时：
   - VS Code 触发 `auto-researcher.showSessionAnalytics` 命令
   - **由于没有明确指定 arguments，VS Code 自动将 TreeItem 本身作为第一个参数传递**
   - 这是 VS Code TreeView 的默认行为

3. 原有的命令注册代码：
   ```typescript
   vscode.commands.registerCommand('auto-researcher.showSessionAnalytics', 
       (sessionId: string) => researchCommands.showSessionAnalytics(sessionId)
   );
   ```
   - 期望接收字符串
   - 但实际接收到 TreeItem 对象
   - 导致 sessionId 参数是整个对象

### 🛠️ 修复方案

#### [Step 5: 修复命令参数处理]

**Action**: 修改命令注册逻辑，支持从 TreeItem 对象中提取 sessionId

**修改内容**:
```typescript
// Before (extension.ts:2056-2058):
const showSessionAnalyticsCommand = vscode.commands.registerCommand('auto-researcher.showSessionAnalytics', 
    (sessionId: string) => researchCommands.showSessionAnalytics(sessionId)
);

// After:
const showSessionAnalyticsCommand = vscode.commands.registerCommand('auto-researcher.showSessionAnalytics', 
    (arg: any) => {
        // Handle both cases: called from menu (TreeItem object) or programmatically (string)
        let sessionId: string;
        if (typeof arg === 'string') {
            sessionId = arg;
        } else if (arg && typeof arg === 'object' && arg.sessionId) {
            // Extract sessionId from TreeItem object
            sessionId = arg.sessionId;
        } else {
            console.error('[showSessionAnalyticsCommand] Invalid argument:', arg);
            vscode.window.showErrorMessage('Cannot show analytics: Invalid session');
            return;
        }
        researchCommands.showSessionAnalytics(sessionId);
    }
);
```

**为什么这样修复**:
- 支持**两种调用方式**：
  1. 从 TreeView 右键菜单调用 → 传入 TreeItem 对象 → 提取 sessionId
  2. 从代码中直接调用 → 传入字符串 sessionId → 直接使用
- 添加类型检查和错误处理
- 兼容性更好，不会破坏现有功能

// ManuscriptDocumentProvider.ts:195
private extractVersion(report: any): number {
    // Try to extract from title first
    if (report.title) {
        const match = report.title.match(/v(\d+)/i) || ...
        if (match) {
            return parseInt(match[1], 10);
        }

**结果**:
- TreeView 显示的版本号（如 v3）通过 ID hash 计算得出
- ManuscriptDocumentProvider 用同样的逻辑计算版本号
- 但由于 hash 的不确定性，可能产生不同的结果（如 98）
- 导致 "Version 98 not found" 错误

### 🛠️ 修复方案

#### [Step 7: 统一版本号提取逻辑]

**Action**: 修改两个 `extractVersion` 方法，优先使用 `report.version` 字段

**修改 1**: `ManuscriptDocumentProvider.ts`
```typescript
// Before:
private extractVersion(report: any): number {
    // Try to extract from title first
    if (report.title) {
        const match = report.title.match(/v(\d+)/i) || report.title.match(/version\s*(\d+)/i);
        if (match) {
            return parseInt(match[1], 10);
        }
    }
    // Fallback: use report ID hash
    return report.id ? parseInt(report.id.slice(-4), 16) % 100 : 1;
}

// After:
private extractVersion(report: any): number {
    // First, try to use the version field directly ← 优先使用
    if (typeof report.version === 'number') {
        return report.version;
    }
    
    // Try to extract from title
    if (report.title) {
        const match = report.title.match(/v(\d+)/i) || report.title.match(/version\s*(\d+)/i);
        if (match) {
            return parseInt(match[1], 10);
        }
    }
    
    // Fallback: use report ID hash
    return report.id ? parseInt(report.id.slice(-4), 16) % 100 : 1;
}
```
