
### Hypothesis
**错误原因:** 后端不使用自定义的 `/threads` 端点

**发现:**
1. 检查 `backend/API_DOCUMENTATION.md` 发现后端使用 LangGraph API 标准
2. LangGraph 使用不同的 API 模式：
   - ❌ 不存在: `POST /threads`
   - ✅ 正确: `POST /agent/invoke` (with `thread_id` in config)
   - ✅ 正确: `GET /agent/state/{thread_id}`

**核心区别:**
- **LangGraph 模式**: 通过 `config.configurable.thread_id` 传递线程 ID
- **原计划模式**: 独立的 `/threads` 端点管理

### Root Cause Analysis
**根本原因:** 违反了 GEMINI.md 的 "Principle 0: API Contract First"

- ❌ **错误做法**: 未查看 `openapi.yaml` 就假设 API 结构
- ✅ **正确做法**: 应该先查看后端 API 契约，再设计前端函数

**教训:**
> 在任何涉及前后端交互的功能开发中，**必须先查看 `openapi.yaml`** 或后端 API 文档，理解真实的 API 结构。

### Fix Attempt #1
**策略:** 重新设计 API 客户端函数，使其符合 LangGraph API 标准

**需要修改的函数:**
1. ~~`createThread()`~~ → 删除（LangGraph 自动管理线程）
2. `startResearch()` → 修改为调用 `POST /agent/invoke`
3. `getThreadState()` → 保持（但调整路径为 `/agent/state/{thread_id}`）

## 代码实现 (Implementation)

### 添加的接口定义

```typescript
// 1. 创建研究线程
export interface CreateThreadRequest {
  metadata?: {
    research_topic?: string;
    [key: string]: any;
  };
}

export interface ThreadResponse {
  thread_id: string;
  created_at: string;
  metadata: any;
}

export async function createThread(
  request: CreateThreadRequest
): Promise<ThreadResponse> {
  try {
    const response = await axios.post(`${API_BASE_URL}/threads`, request);
    return response.data;
  } catch (error: any) {
    console.error('Error creating thread:', error);
    throw error;
  }
}

// 2. 启动研究任务
export interface StartResearchRequest {
  assistant_id: string;  // "agent"
  input: {
    messages: Array<{
      role: string;
      content: string;
    }>;
  };
  stream_mode: string[];  // ["values", "updates"]
}

export async function startResearch(
  threadId: string,
  topic: string
): Promise<string> {
  try {
    const request: StartResearchRequest = {
      assistant_id: "agent",
      input: {
        messages: [
          {
            role: "user",
            content: `Please research: ${topic}`
          }
        ]
      },
      stream_mode: ["values", "updates"]
    };
    
    const response = await axios.post(
      `${API_BASE_URL}/threads/${threadId}/runs/stream`,
      request
    );
    return response.data.run_id || response.data.id;
  } catch (error: any) {
    console.error('Error starting research:', error);
    throw error;
  }
}

// 3. 获取线程状态
export interface ThreadState {
  values: AgentState;
  next: string[];
  metadata: any;
}

export async function getThreadState(
  threadId: string
): Promise<ThreadState> {
  try {
    const response = await axios.get(
      `${API_BASE_URL}/threads/${threadId}/state`
    );
    return response.data;
  } catch (error: any) {
    console.error('Error fetching thread state:', error);
    throw error;
  }
}
```

## 调试快照 #2 (Debugging Snapshot #2)

### Error
```
AttributeError: 'Langfuse' object has no attribute 'trace'
```

### Root Cause
**问题分析:**
1. 多个文件独立导入和初始化 Langfuse
2. Langfuse 2.x API 使用 `start_observation()` 而不是 `trace()`
3. 当 Langfuse 凭证缺失时，对象创建但方法不可用

### Fix Strategy
**架构改进:** 创建统一的 Langfuse 初始化模块

**优点:**
- 单一真相来源（Single Source of Truth）
- 安全的初始化检查
- 优雅降级（No-op 模式）
- 避免重复的初始化逻辑

### Implementation
**新建文件:** `backend/src/agent/langfuse_manager.py`

**核心功能:**
1. `LangfuseManager` 类 - 单例模式管理 Langfuse 实例
2. `initialize()` - 检查环境变量并初始化
3. `trace()` - 适配方法（暂时返回 No-op）
4. `NoOpTrace` / `NoOpSpan` - 无操作上下文管理器

**迁移文件:**
- ✅ `backend/src/agent/app.py`
- ✅ `backend/src/agent/hitl_nodes.py`
- ✅ `backend/src/agent/document_utils.py`
- ✅ `backend/src/agent/graph.py`

**Note:** 当前 `trace()` 方法返回 NoOpTrace，因为 Langfuse 2.x API 兼容性问题。可观测性集成将在后续优先级更高时完善。
