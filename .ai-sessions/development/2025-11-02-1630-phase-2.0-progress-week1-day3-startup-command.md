#### 2. 重构启动逻辑 (`extension.ts` lines 920-1074)

**Before (Mock):**
```typescript

setTimeout(() => { /* mock progress */ }, 1000);

```

**After (Real API):**
```typescript
const threadId = generateThreadId(); // UUID v4

// Start research
await invokeAgent(threadId, topic);

// Poll for progress (every 5 seconds)
const pollIntervalId = setInterval(async () => {
    const state = await getThreadState(threadId);
    
    // Interpret state to show progress
    if (state.search_queries?.length > 0) {
        progress = 40;
        statusMessage = `📝 Generated ${state.search_queries.length} queries`;
    }
    
    if (state.literature_abstracts?.length > 0) {
        progress = 60;
        statusMessage = `📚 Found ${state.literature_abstracts.length} papers`;
    }
    
    if (state.report?.length > 100) {
        progress = 100;
        clearInterval(pollIntervalId);
        // Show completion
    }
    
    // Handle HITL waiting state
    if (lastMessage.includes('Waiting for user approval')) {
        vscode.window.showWarningMessage('⏸️ Research paused: HITL');
    }
    
    panel.webview.postMessage({command, message, progress});
}, 5000); // Poll every 5 seconds

// Clean up on panel close
panel.onDidDispose(() => clearInterval(pollIntervalId));
```

**Key Features:**
1. ✅ Real UUID v4 thread ID
2. ✅ Actual backend API calls
3. ✅ Progress interpretation from state
4. ✅ HITL pause detection
5. ✅ Timeout protection (max 10 minutes)
6. ✅ Proper cleanup on panel close
