### Step 1: 数据库架构设计

#### 1.1 分析需求

根据 ROADMAP.md Phase 3.5.1 要求：

**核心表设计**：
```sql
-- 1. sessions: 研究会话元数据
CREATE TABLE sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    thread_id UUID NOT NULL UNIQUE,  -- 关联 LangGraph checkpoints
    title VARCHAR(500) NOT NULL,
    research_topic TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    status VARCHAR(50) DEFAULT 'active',  -- active, completed, archived
    tags TEXT[],  -- 标签数组
    notes TEXT
);

-- 2. papers: 收藏的论文
CREATE TABLE papers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
    title VARCHAR(1000) NOT NULL,
    authors TEXT[],
    abstract TEXT,
    doi VARCHAR(255),
    arxiv_id VARCHAR(50),
    url TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    metadata JSONB  -- 扩展字段
);

-- 3. reports: 生成的研究报告
CREATE TABLE reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    format VARCHAR(50) DEFAULT 'markdown',  -- markdown, pdf, html
    version INT DEFAULT 1,
    created_at TIMESTAMP DEFAULT NOW(),
    metadata JSONB
);

-- 4. session_events: 会话事件日志
CREATE TABLE session_events (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
    event_type VARCHAR(100) NOT NULL,  -- query_generated, paper_retrieved, report_created
    event_data JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_sessions_thread_id ON sessions(thread_id);
CREATE INDEX idx_sessions_status ON sessions(status);
CREATE INDEX idx_papers_session_id ON papers(session_id);
CREATE INDEX idx_reports_session_id ON reports(session_id);
CREATE INDEX idx_events_session_id ON session_events(session_id);
CREATE INDEX idx_events_type ON session_events(event_type);
```

**设计决策**：
- ✅ 使用 UUID 作为主键（与 LangGraph checkpointer 一致）
- ✅ `thread_id` 作为外键关联 LangGraph 状态
- ✅ JSONB 存储扩展元数据（灵活性）
- ✅ 级联删除（数据一致性）
- ✅ 时间戳审计（created_at, updated_at）

## 问题与解决方案

### 问题 1: SQLAlchemy `metadata` 属性名冲突 ✅ 已解决

**时间**: 2025-10-12 15:57

**错误信息**:
```
sqlalchemy.exc.InvalidRequestError: Attribute name 'metadata' is reserved when using the Declarative API.
```

**问题描述**:
在 `Paper` 和 `Report` 模型中使用了 `metadata` 作为 JSONB 列名，但这与 SQLAlchemy Declarative API 的内部保留属性冲突。`metadata` 属性用于存储表的元数据信息。

**解决方案**:
将所有 `metadata` 列重命名为 `extra_metadata`：

1. **models.py 修改**:
   ```python
   # Paper 模型
   extra_metadata = Column(JSONB, default=dict, comment="Additional paper metadata")
   
   # Report 模型
   extra_metadata = Column(JSONB, default=dict, comment="Report generation metadata")
   ```

2. **db_manager.py 同步更新**:
   - `add_paper()` 参数: `metadata` → `extra_metadata`
   - `create_report()` 参数: `metadata` → `extra_metadata`

3. **to_dict() 方法更新**:
   ```python
   'extra_metadata': self.extra_metadata or {}
   ```

**验证**:
```bash
docker compose restart langgraph-api
# 检查服务是否正常启动，无 SQLAlchemy 错误
```

**参考**:
- SQLAlchemy 保留属性: https://docs.sqlalchemy.org/en/20/orm/mapping_api.html#sqlalchemy.orm.DeclarativeBase.metadata
- 类似问题: https://github.com/sqlalchemy/sqlalchemy/discussions/7805

## 关键学习点总结

### 1. SQLAlchemy 保留属性
- ❌ **问题**: `metadata` 是 Declarative API 的保留属性
- ✅ **解决方案**: 重命名为 `extra_metadata`
- 📁 **影响文件**: models.py, db_manager.py

### 2. ORM 对象生命周期
- ❌ **问题**: 不能在 `with` 上下文外返回 ORM 对象（会话已分离）
- ✅ **解决方案**: 在上下文内调用 `.to_dict()` 转换为字典
- 📁 **影响函数**: 所有 CRUD 函数的返回类型改为 `Dict[str, Any]`

### 3. FastAPI TestClient 测试
- 使用 `TestClient` 进行集成测试
- 自动处理数据库事务
- 支持完整的请求/响应周期测试

### 4. 数据库表自动创建
- `Base.metadata.create_all(bind=engine)` 自动创建表
- 幂等操作，安全调用多次
- 在 `@app.on_event("startup")` 中调用
