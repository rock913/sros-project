## 环境变量配置 (Environment Variables)

### 必需变量

在项目根目录创建或更新 `.env` 文件：

```bash
# LangFuse 可观测性配置
LANGFUSE_PUBLIC_KEY=pk-lf-xxxxxxxxxxxxxxxxxxxxxxxx
LANGFUSE_SECRET_KEY=sk-lf-xxxxxxxxxxxxxxxxxxxxxxxx
LANGFUSE_HOST=http://your-langfuse-host:3000
```

### 变量说明

| 变量名 | 说明 | 示例值 | 是否必需 |
|--------|------|--------|---------|
| `LANGFUSE_PUBLIC_KEY` | LangFuse 公钥 | `pk-lf-abc123...` | ✅ 是 |
| `LANGFUSE_SECRET_KEY` | LangFuse 密钥 | `sk-lf-xyz789...` | ✅ 是 |
| `LANGFUSE_HOST` | LangFuse 服务地址 | `http://localhost:3000` | ⚠️ 可选 (默认 localhost:3000) |

### 获取 LangFuse 密钥

#### 方法 1: 从已部署的 LangFuse 服务获取

1. 登录您部署的 LangFuse Dashboard
2. 进入 **Settings** → **API Keys**
3. 创建新的 API Key 或复制现有密钥
4. 复制 `Public Key` 和 `Secret Key`

#### 方法 2: 使用本地 LangFuse (测试环境)

如果您还没有部署 LangFuse，可以使用 Docker Compose 快速启动：

```bash
# 创建 langfuse 目录
mkdir -p langfuse && cd langfuse

# 下载 docker-compose.yml
curl -o docker-compose.yml https://raw.githubusercontent.com/langfuse/langfuse/main/docker-compose.yml

# 启动 LangFuse
docker-compose up -d

# 访问 http://localhost:3000
# 注册账号并创建 API Key
```

### 3. 验证 LangFuse 连接

创建测试脚本 `test_langfuse.py`:

```python
from langfuse import Langfuse
import os

# 初始化 LangFuse 客户端
langfuse = Langfuse(
    public_key=os.getenv("LANGFUSE_PUBLIC_KEY"),
    secret_key=os.getenv("LANGFUSE_SECRET_KEY"),
    host=os.getenv("LANGFUSE_HOST", "http://localhost:3000")
)

# 创建测试 trace
trace = langfuse.trace(
    name="Connection Test",
    input={"message": "Testing LangFuse connection from backend"},
    tags=["test", "connection"]
)

# 更新 trace 输出
trace.update(output={"status": "success", "message": "LangFuse connection verified!"})

print("✅ Test trace created successfully!")
print(f"📊 Visit your LangFuse Dashboard at {os.getenv('LANGFUSE_HOST', 'http://localhost:3000')} to view the trace.")
```

运行测试：

```bash
# 在容器内运行测试
docker exec -it langgraph-api python /deps/backend/test_langfuse.py
```

**预期输出**:
```
✅ Test trace created successfully!
📊 Visit your LangFuse Dashboard at http://your-langfuse-host:3000 to view the trace.
```

## LangFuse Dashboard 使用指南

### 1. Traces 页面

**功能**: 查看所有 trace 列表

**操作**:
- 点击 **Traces** 进入主页面
- 使用搜索框过滤 trace (例如: 搜索 session_id)
- 按时间、tags、status 筛选
- 点击 trace 查看详情

---

### 2. Trace 详情页面

**功能**: 查看单个 trace 的完整信息

**关键信息**:
- **Timeline**: Trace 和 Span 的时间线 (可视化性能瓶颈)
- **Input**: 输入参数
- **Output**: 输出结果
- **Metadata**: session_id, user_id 等元数据
- **Tags**: 用于分类和搜索
- **Spans**: 子操作详情 (例如: LLM 调用, 数据库操作)

**性能分析**:
- 查看每个 span 的耗时
- 识别性能瓶颈 (例如: LLM 调用时间过长)
- 优化建议

---

### 3. Sessions 页面

**功能**: 按 session_id 分组查看 trace

**用途**:
- 跟踪单个用户会话的完整流程
- 分析用户行为路径
- 识别异常会话

---

### 4. Metrics 页面

**功能**: 查看聚合统计数据

**关键指标**:
- **Total Traces**: 总 trace 数量
- **Average Duration**: 平均耗时
- **Error Rate**: 错误率
- **Token Usage**: LLM token 消耗 (如果配置)
- **Cost**: 估算成本 (如果配置)

## 常见问题 (Troubleshooting)

### 问题 1: trace 未出现在 Dashboard

**可能原因**:
1. 环境变量配置错误
2. LangFuse 服务未启动
3. 网络连接问题

**解决方法**:
```bash
# 1. 检查环境变量
docker exec -it langgraph-api env | grep LANGFUSE

# 2. 检查 LangFuse 服务状态
docker ps | grep langfuse
# 或访问 http://your-langfuse-host:3000

# 3. 查看后端日志
docker-compose -f docker-compose-dev.yml logs -f langgraph-api | grep -i langfuse

# 4. 测试连接
docker exec -it langgraph-api python test_langfuse.py
```

---

### 问题 2: "Module not found: langfuse"

**原因**: LangFuse SDK 未安装

**解决方法**:
```bash
# 重新构建容器
docker-compose -f docker-compose-dev.yml build langgraph-api

# 或手动安装 (临时)
docker exec -it langgraph-api pip install langfuse
```

---

### 问题 3: "Authentication failed"

**原因**: 密钥错误或过期

**解决方法**:
```bash
# 1. 验证密钥格式
echo $LANGFUSE_PUBLIC_KEY  # 应以 pk-lf- 开头
echo $LANGFUSE_SECRET_KEY  # 应以 sk-lf- 开头

# 2. 重新生成密钥
# 登录 LangFuse Dashboard → Settings → API Keys → Create New Key

# 3. 更新 .env 文件
nano .env

# 4. 重启服务
docker-compose -f docker-compose-dev.yml restart langgraph-api
```

---

### 问题 4: trace 数据不完整

**可能原因**:
1. 代码异常导致 `trace.update()` 未执行
2. 网络中断导致上传失败

**解决方法**:
```bash
# 1. 查看后端日志中的异常
docker-compose -f docker-compose-dev.yml logs -f langgraph-api

# 2. 添加错误处理
# 在关键代码段添加 try-except:
try:
    trace.update(output={...})
except Exception as e:
    logger.error(f"Failed to update trace: {e}")

# 3. 检查网络连接
docker exec -it langgraph-api curl -I http://your-langfuse-host:3000
```

---

### 问题 5: Dashboard 性能慢

**原因**: Trace 数据量过大

**解决方法**:
1. **配置采样率** (Sampling):
   ```python
   from langfuse import Langfuse
   langfuse = Langfuse(sample_rate=0.1)  # 仅采集 10% 的 trace
   ```

2. **定期清理历史数据**:
   - 进入 LangFuse Dashboard
   - Settings → Data Retention
   - 配置自动清理策略 (例如: 保留 30 天)

3. **按需采集**:
   ```python
   import os
   # 仅在开发环境启用 trace
   if os.getenv("ENVIRONMENT") == "development":
       trace = langfuse.trace(...)
   ```

## 性能优化建议 (Performance Optimization)

### 1. 异步上传

LangFuse SDK 默认异步上传，无需额外配置。但可以调整参数：

```python
from langfuse import Langfuse

langfuse = Langfuse(
    flush_at=10,        # 每 10 个 trace 批量上传
    flush_interval=1.0  # 或每 1 秒上传一次
)
```

---

### 2. 减少 Payload 大小

**优化前** (不推荐):
```python
trace = langfuse.trace(
    name="Report Synthesis",
    input={
        "full_context": rag_context,  # 可能有几 MB
        "full_prompt": prompt         # 可能有几十 KB
    }
)
```

**优化后** (推荐):
```python
trace = langfuse.trace(
    name="Report Synthesis",
    input={
        "context_length": len(rag_context),
        "prompt_preview": prompt[:500] + "..." if len(prompt) > 500 else prompt
    }
)
```

---

### 3. 按需采集

```python
import os

# 仅在需要时启用详细 trace
ENABLE_DETAILED_TRACE = os.getenv("ENABLE_DETAILED_TRACE", "false") == "true"

if ENABLE_DETAILED_TRACE:
    trace = langfuse.trace(...)
    # 详细采集
else:
    # 仅采集关键指标
    trace = langfuse.trace(name="...", input={"session_id": session_id})
```
