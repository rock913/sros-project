## 🎯 Testing Prerequisites

### Backend Requirements
1. **Backend Running**: Ensure backend server is running on `http://localhost:8001`
   ```bash
   cd backend
   make dev
   ```

2. **Test Data**: Verify you have at least one completed research session with:
   - Research topic
   - Papers (at least 1-2)
   - Report/Manuscript

3. **API Health Check**:
   ```bash
   curl http://localhost:8001/health
   curl http://localhost:8001/sessions
   ```

### Extension Setup
1. **Open VS Code**: Open the `vscode-extension` folder
2. **Install Dependencies**: `npm install` (if not done)
3. **Compile**: `npm run compile` (should complete without errors ✅)
4. **Launch**: Press **F5** to open Extension Development Host

## 🐛 Known Issues / Limitations

### Current Limitations
1. **Pagination**: "Load more" button is placeholder (not implemented)
2. **Real-time Updates**: View doesn't auto-refresh when sessions change
3. **PDF Export**: Depends on backend implementation
4. **BibTeX API**: Assumes `/papers/{id}/bibtex` endpoint exists

### Future Enhancements
1. **Auto-refresh**: Poll for new sessions every 30s
2. **WebSocket**: Real-time session status updates
3. **Search**: Server-side search for large datasets
4. **Filters**: More filter options (date range, paper count, etc.)
5. **Sorting**: Sort sessions by date, status, paper count
6. **Bulk Actions**: Select multiple sessions, bulk export
