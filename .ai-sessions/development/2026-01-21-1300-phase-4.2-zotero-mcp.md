# Session: Phase 4.2 - Tool Migration (Zotero)

**Date:** 2026-01-21
**Architect:** GitHub Copilot
**Phase:** 4.2 (MCP Infrastructure & Hexagonal Migration)

## Objective
Migrate the `Zotero` reference management capability to the MPA/MCP architecture.

## Architecture

### 1. Domain Layer (`agent.domain`)
- **Schema**: Reuse `Paper` from `agent.domain.schemas.paper`.
- **Protocol**: `ReferenceManager` in `agent.domain.ports.reference_manager.py`.
    - Method: `save_paper(paper: Paper) -> str` (Returns Item Key or Confirmation)

### 2. Infrastructure Layer (`agent.infrastructure`)
- **Adapter**: `ZoteroAdapter` in `agent.infrastructure.tools.zotero_adapter.py`.
    - Implementation: Use `pyzotero`. Requires Env Vars (`ZOTERO_LIBRARY_ID`, `ZOTERO_API_KEY`).
    - Logic: Map Domain `Paper` -> Zotero Item Template -> API Call.

### 3. MCP Layer (`agent.infrastructure.mcp`)
- **Tool**: `get_zotero_save_tool()` in `agent.infrastructure.mcp.tools.zotero.py`.
    - Wrapper around the Adapter.

## Execution Steps
1.  **Design Protocol**: `ReferenceManager`.
2.  **Implement Adapter**: `ZoteroAdapter`.
3.  **Create MCP Wrapper**: `McpTool`.
4.  **Register**: Update `entrypoint.py`.
