#### 2. WebSocket 会话 Trace 初始化 (Lines 1318-1338)

**变更**:
```python
await websocket.accept()
thread_id = None
session_id = None
trace = None  # LangFuse trace for WebSocket session

try:
    # 1. Receive initial request
    data = await websocket.receive_json()
    
    # 2. Generate or use provided thread_id
    thread_id = data.get("thread_id") or str(uuid4())
    messages = data.get("messages", [])
    
    # 3. Create LangFuse trace for entire WebSocket session
    trace = langfuse.trace(
        name="WebSocket Research Session",
        input={
            "thread_id": thread_id,
            "message_count": len(messages),
            "has_existing_thread": bool(data.get("thread_id"))
        },
        tags=["websocket", "streaming", "research_session"]
    )
```

**采集数据**:
- **Input**:
  - `thread_id`: 会话线程 ID
  - `message_count`: 初始消息数量
  - `has_existing_thread`: 是否恢复已有会话
- **Tags**: websocket, streaming, research_session

#### 4. 文档更新 Span (Lines 1432-1467)

**变更**:
```python
# 📝 DOCUMENT UPDATE: Check for partial report updates
current_report = state_update.get("report") or state_update.get("final_report", "")
if current_report and current_report != last_report_version:
    # Create span for document update
    doc_span = trace.span(
        name="WebSocket Document Update",
        input={
            "node": node_name,
            "old_length": len(last_report_version),
            "new_length": len(current_report)
        },
        tags=["websocket", "document_update"]
    )
    
    # Generate incremental diff
    diffs = differ.generate_paragraph_diff(last_report_version, current_report)
    
    # Send only changed paragraphs to frontend
    for diff in diffs:
        if diff["action"] != "unchanged":
            # ... send to frontend ...
            pass
    
    # End document update span
    doc_span.end(output={
        "diff_count": len(diffs),
        "changed_paragraphs": len([d for d in diffs if d["action"] != "unchanged"])
    })
    
    # Update last known version
    last_report_version = current_report
```

**采集数据**:
- **Input**:
  - `node`: 触发更新的节点名称
  - `old_length`: 旧文档长度
  - `new_length`: 新文档长度
- **Output**:
  - `diff_count`: 总 diff 数量
  - `changed_paragraphs`: 实际变更的段落数
- **Tags**: websocket, document_update

**用途**:
- 追踪实时文档流式推送的性能
- 分析文档生成速度和变更频率
- 监控前端接收到的增量更新数量

## Trace 层级结构 (Trace Hierarchy)

```
WebSocket Research Session (Trace)
├── Input: thread_id, message_count, has_existing_thread
├── Metadata: session_id, research_topic
├── Spans:
│   ├── WebSocket Document Update (Span 1)
│   │   ├── Input: node, old_length, new_length
│   │   └── Output: diff_count, changed_paragraphs
│   ├── WebSocket Document Update (Span 2)
│   │   └── ...
│   ├── WebSocket HITL Request (Span 1)
│   │   ├── Input: request_id, decision_type, timeout_seconds
│   │   └── Output: sent_to_client, timestamp
│   └── WebSocket HITL Request (Span 2)
│       └── ...
└── Output (完成): status, paper_count, has_report, word_count
    OR (中断): disconnect_reason
    OR (错误): error, traceback
```

## 数据流分析 (Data Flow Analysis)

### 1. 正常研究流程

```
1. WebSocket 连接建立
   └── Trace: "WebSocket Research Session" created

2. 用户发送研究主题
   └── Trace updated: session_id, research_topic

3. Agent 开始生成报告
   └── Span: "WebSocket Document Update" (第 1 次)
       ├── Input: node="retrieve_and_synthesize_report", old_length=0, new_length=500
       └── Output: diff_count=5, changed_paragraphs=5

4. Agent 继续生成报告
   └── Span: "WebSocket Document Update" (第 2 次)
       ├── Input: node="retrieve_and_synthesize_report", old_length=500, new_length=1200
       └── Output: diff_count=10, changed_paragraphs=7

5. Agent 请求 HITL 审核
   └── Span: "WebSocket HITL Request"
       ├── Input: request_id="hitl_report_revision_abc123", decision_type="report_revision"
       └── Output: sent_to_client=true

6. 用户审核并批准
   (此时 HITL 响应通过 /agent/hitl/respond 端点，不在 WebSocket trace 中)

7. Agent 完成研究
   └── Trace updated:
       └── Output: status="completed", paper_count=15, has_report=true, word_count=1200
```

---

### 2. 用户中断流程

```
1-4. (同正常流程)

5. 用户关闭浏览器标签页
   └── WebSocketDisconnect 异常触发
       └── Trace updated:
           ├── status="interrupted"
           ├── status_message="Client disconnected"
           └── output.disconnect_reason="WebSocket connection lost"
```

---

### 3. 错误场景流程

```
1-3. (同正常流程)

4. Agent 在生成报告时遇到 LLM API 错误
   └── Exception 触发
       └── Trace updated:
           ├── status="error"
           ├── status_message="ServiceUnavailableError: Gemini API rate limit exceeded"
           └── output.error="ServiceUnavailableError: Gemini API rate limit exceeded"
           └── output.traceback="Traceback (most recent call last):\n  ..."
```
