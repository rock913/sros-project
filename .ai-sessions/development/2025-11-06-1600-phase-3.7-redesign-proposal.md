#### 2.3 CodeLens 集成

在 Manuscript 文档中提供 CodeLens 操作：

```markdown
## References
[1] Smith, J. (2024)...  ← [View Paper] [Copy BibTeX] [Download PDF]
[2] Johnson, K. (2024)... ← [View Paper] [Copy BibTeX] [Download PDF]
```

**实现**:
```typescript
class ManuscriptCodeLensProvider implements vscode.CodeLensProvider {
  provideCodeLenses(document: vscode.TextDocument) {
    const codeLenses: vscode.CodeLens[] = [];
    
    // 检测引用行: [1] Author (Year)
    const refRegex = /^\[(\d+)\]\s+(.+)$/gm;
    let match;
    
    while ((match = refRegex.exec(document.getText())) !== null) {
      const line = document.positionAt(match.index).line;
      const range = new vscode.Range(line, 0, line, 0);
      
      codeLenses.push(
        new vscode.CodeLens(range, {
          title: "$(eye) View Paper",
          command: 'auto-researcher.viewPaper',
          arguments: [match[1]]
        }),
        new vscode.CodeLens(range, {
          title: "$(clippy) Copy BibTeX",
          command: 'auto-researcher.copyBibTeX',
          arguments: [match[1]]
        }),
        new vscode.CodeLens(range, {
          title: "$(cloud-download) Download PDF",
          command: 'auto-researcher.downloadPDF',
          arguments: [match[1]]
        })
      );
    }
    
    return codeLenses;
  }
}
```

### 3. Version Management

#### 3.1 版本对比

**场景**: 用户想比较 v2 和 v3 的差异

**操作**:
```
1. 右键点击 "Manuscript v3"
2. 选择 "Compare with Previous Version..."
3. VS Code 打开 Diff View:
   ┌───────────────────┬───────────────────┐
   │ v2 (Old)         │ v3 (New)          │
   ├───────────────────┼───────────────────┤
   │ ## Key Findings  │ ## Key Findings   │
   │ 1. Accuracy [1]  │ 1. Accuracy [1][2]│ ← 新增引用
   │                  │ 2. Outcomes [3]   │ ← 新增发现
   └───────────────────┴───────────────────┘
```

**实现**:
```typescript
const compareVersionsCommand = vscode.commands.registerCommand(
  'auto-researcher.compareVersions',
  async (sessionId: string, versionNew: number, versionOld: number) => {
    const uriOld = vscode.Uri.file(`research-session-${sessionId}-v${versionOld}.md`);
    const uriNew = vscode.Uri.file(`research-session-${sessionId}-v${versionNew}.md`);
    
    await vscode.commands.executeCommand(
      'vscode.diff',
      uriOld,
      uriNew,
      `Manuscript v${versionOld} ↔ v${versionNew}`
    );
  }
);
```

## 6. 技术实现

### 6.1 TreeDataProvider 架构

```typescript
export class ResearchSessionsTreeProvider implements vscode.TreeDataProvider<TreeItem> {
  private _onDidChangeTreeData = new vscode.EventEmitter<TreeItem | undefined>();
  readonly onDidChangeTreeData = this._onDidChangeTreeData.event;

  constructor(private context: vscode.ExtensionContext) {}

  refresh(): void {
    this._onDidChangeTreeData.fire(undefined);
  }

  getTreeItem(element: TreeItem): vscode.TreeItem {
    return element;
  }

  async getChildren(element?: TreeItem): Promise<TreeItem[]> {
    if (!element) {
      // Root level: Return sessions
      return this.getSessions();
    }

    if (element.contextValue === 'research-session') {
      // Session level: Return manuscripts, papers, analytics
      return this.getSessionChildren(element.sessionId);
    }

    if (element.contextValue === 'papers-group') {
      // Papers group: Return paper list
      return this.getPapers(element.sessionId);
    }

    return [];
  }

  private async getSessions(): Promise<SessionTreeItem[]> {
    const response = await getSessionsList({ limit: 100 });
    return response.sessions.map(s => new SessionTreeItem(s));
  }

  private async getSessionChildren(sessionId: string): Promise<TreeItem[]> {
    const details = await getSessionDetailsV2(sessionId);
    
    return [
      new ManuscriptGroupTreeItem(sessionId, details.reports),
      new PapersGroupTreeItem(sessionId, details.papers.length),
      new AnalyticsTreeItem(sessionId, details.stats)
    ];
  }

  private async getPapers(sessionId: string): Promise<PaperTreeItem[]> {
    const details = await getSessionDetailsV2(sessionId);
    return details.papers.map((p, i) => new PaperTreeItem(p, i + 1));
  }
}
```

### 6.2 TreeItem 类定义

```typescript
class SessionTreeItem extends vscode.TreeItem {
  constructor(public session: Session) {
    super(session.research_topic, vscode.TreeItemCollapsibleState.Collapsed);
    
    this.tooltip = `Created: ${formatDate(session.created_at)}\nPapers: ${session.paper_count}\nStatus: ${session.status}`;
    this.iconPath = new vscode.ThemeIcon('notebook');
    this.contextValue = 'research-session';
    this.sessionId = session.id;
    
    // Status indicator
    const statusIcons = {
      'completed': '✅',
      'in_progress': '🔄',
      'failed': '❌',
      'paused': '⏸️'
    };
    this.description = `${statusIcons[session.status]} ${session.paper_count} papers`;
  }
}

class ManuscriptVersionTreeItem extends vscode.TreeItem {
  constructor(
    public sessionId: string,
    public version: number,
    public isLatest: boolean,
    public content: string
  ) {
    const label = `Manuscript v${version}${isLatest ? ' (Latest)' : ''}`;
    super(label, vscode.TreeItemCollapsibleState.None);
    
    this.iconPath = new vscode.ThemeIcon('file-text');
    this.contextValue = 'manuscript-version';
    
    // Click to open
    this.command = {
      command: 'auto-researcher.openManuscript',
      title: 'Open Manuscript',
      arguments: [sessionId, version]
    };
    
    const wordCount = content.split(/\s+/).length;
    this.description = `${wordCount} words`;
  }
}

class PaperTreeItem extends vscode.TreeItem {
  constructor(public paper: Paper, public refNumber: number) {
    super(paper.title, vscode.TreeItemCollapsibleState.None);
    
    this.tooltip = `${paper.title}\n\nAuthors: ${paper.authors.join(', ')}\nYear: ${paper.year}\nDOI: ${paper.doi || 'N/A'}`;
    this.iconPath = new vscode.ThemeIcon('file-pdf');
    this.contextValue = 'paper';
    
    this.label = `[${refNumber}] ${paper.authors[0]?.split(',')[0] || 'Unknown'} et al. (${paper.year})`;
    this.description = paper.publication || 'Unknown Journal';
    
    // Click to open paper details
    this.command = {
      command: 'auto-researcher.openPaper',
      title: 'Open Paper',
      arguments: [paper.id]
    };
  }
}
```

### 6.3 Document Provider

```typescript
class ManuscriptDocumentProvider implements vscode.TextDocumentContentProvider {
  private _onDidChange = new vscode.EventEmitter<vscode.Uri>();
  readonly onDidChange = this._onDidChange.event;

  async provideTextDocumentContent(uri: vscode.Uri): Promise<string> {
    // URI format: research-manuscript://session-{id}/v{version}
    const match = uri.path.match(/session-(.+)\/v(\d+)/);
    if (!match) {
      throw new Error('Invalid URI format');
    }

    const [, sessionId, version] = match;
    const details = await getSessionDetailsV2(sessionId);
    const report = details.reports[parseInt(version) - 1];

    if (!report) {
      throw new Error(`Version ${version} not found`);
    }

    // Generate markdown with frontmatter
    return this.generateManuscriptMarkdown(details.session, report, version);
  }

  private generateManuscriptMarkdown(
    session: Session,
    report: Report,
    version: string
  ): string {
    return `---
session_id: ${session.id}
session_topic: ${session.research_topic}
version: ${version}
created_at: ${report.created_at}
status: ${session.status}
paper_count: ${session.paper_count}
---

# ${session.research_topic}

**Session ID**: ${session.id}  
**Version**: v${version}  
**Created**: ${formatDateFull(report.created_at)}  
**Papers Referenced**: ${session.paper_count}

---

${report.content}
`;
  }
}
```

## 7. 对比分析

### 方案 A vs 方案 B

| 维度 | 方案 A (Master-Detail) | 方案 B (Tree + Document) |
|------|------------------------|--------------------------|
| **UI复杂度** | 高（Webview HTML/CSS/JS） | 低（原生 Tree View） |
| **开发工作量** | 大（~1,300 行代码） | 中（~600 行代码） |
| **VS Code 集成** | 中等（自定义 Webview） | 高（原生组件） |
| **版本管理** | 需自定义实现 | 利用 VS Code Diff |
| **文档编辑** | 只读（Webview） | 可编辑（Markdown） |
| **性能** | 中等（DOM 渲染） | 高（Tree 懒加载） |
| **用户习惯** | 新交互模式 | 熟悉的 Tree + Editor |
| **扩展性** | 受限于 Webview | 易于扩展（CodeLens 等） |
| **可维护性** | 中（模板字符串） | 高（TypeScript 类） |

### 优缺点对比

**方案 A 优点**:
- ✅ 信息密度高（30/70 布局）
- ✅ 深度集成（Manuscript + Papers 统一视图）
- ✅ 自定义交互（点击引用滚动到 Paper）

**方案 A 缺点**:
- ❌ 开发复杂度高
- ❌ 不利于版本比较
- ❌ Webview 与 VS Code 生态隔离
- ❌ 无法直接编辑内容

**方案 B 优点**:
- ✅ 开发成本低（~50% 代码量）
- ✅ 符合 VS Code 设计语言
- ✅ 原生版本管理（Diff View）
- ✅ 可编辑 Manuscript
- ✅ 性能更好（Tree 懒加载）
- ✅ 易于维护和扩展

**方案 B 缺点**:
- ❌ 信息密度较低（需展开查看）
- ❌ Manuscript 和 Papers 分离显示

```

3. 看到完整的研究报告（带 frontmatter）
4. 引用行上显示 CodeLens:
   [1] Smith (2024) [View Paper] [Copy BibTeX] [Download PDF]
```

```
1. 右键点击 "Manuscript v3"

```
