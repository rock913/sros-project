### 问题诊断

#### 问题 1: 命名格式不统一 🟡

**混用两种格式**:
- 日期格式: `YYYY-MM-DD-description.md` (符合 WORKFLOW_STRATEGY.md 规范)
- 阶段格式: `PHASE_X.Y_TYPE.md` (不符合规范，但便于阶段追踪)

**影响**:
- 无法按时间顺序快速浏览
- 难以判断文件新旧程度
- 违反了 WORKFLOW_STRATEGY.md 的命名规范

#### 问题 2: 阶段文件缺少时间信息 🟡

**示例**:
- `PHASE_3.6_FINAL_COMPLETION_REPORT.md` - 看起来像最终报告，但实际只是 Week 1-2 完成
- `PHASE_3.6_QUICK_REFERENCE.md` - 无法知道哪个版本最新

**影响**:
- 需要用 `ls -lt` 或 `git log` 才能确定文件创建时间
- 文件内容可能与文件名暗示的时间不符

#### 问题 3: 文件数量过多 🟡

**52个文件** 在单一目录下:
- 查找特定文档困难
- 上下文切换成本高
- 容易遗漏重要文档

#### 问题 4: 文档类型混杂 🟢

**多种类型文件混在一起**:
- 日常进度报告 (DAY1_PROGRESS)
- 完成报告 (COMPLETION_REPORT)
- 测试报告 (TEST_REPORT)
- 战略分析 (STRATEGIC_ANALYSIS)
- 快速参考 (QUICK_REFERENCE)
- Debug 记录 (debug-xxx)

## 💡 优化方案

### 方案 A: 增量改进（推荐）⭐

**核心思想**: 保留现有文件，未来新文件遵循统一命名规范

#### A.1 新文件命名规范

**格式**: `YYYY-MM-DD-phase-X.Y-<category>-<description>.md`

**组成部分**:
1. **日期前缀**: `YYYY-MM-DD` (ISO 8601)
2. **阶段标识**: `phase-X.Y` (如 `phase-3.6`)
3. **类别标签**: `<category>` (见下表)
4. **描述**: `<description>` (kebab-case)

**类别标签** (`<category>`):

| 标签 | 用途 | 示例 |
|------|------|------|
| `plan` | 实施计划 | `2025-10-15-phase-3.6-plan-week3-document-collaboration.md` |
| `progress` | 日常进度 | `2025-10-15-phase-3.6-progress-day4-backend-streaming.md` |
| `report` | 完成报告 | `2025-10-20-phase-3.6-report-week3-final-completion.md` |

| `analysis` | 分析文档 | `2025-10-15-phase-3.6-analysis-progress-vs-plan.md` |
| `summary` | 阶段总结 | `2025-10-20-phase-3.6-summary-complete.md` |
| `reference` | 快速参考 | `2025-10-15-phase-3.6-reference-api-endpoints.md` |

**示例**:

- 内部引用需要全局替换

```markdown
### Session File Naming Convention

All session files **must** follow the standardized naming format to ensure chronological traceability:

**Format**: `YYYY-MM-DD-phase-X.Y-<category>-<description>.md`

**Components**:
1. **Date Prefix**: `YYYY-MM-DD` (ISO 8601 format, UTC+0)
2. **Phase Identifier**: `phase-X.Y` (e.g., `phase-3.6`, `phase-4.1`)
3. **Category Tag**: One of: `plan`, `progress`, `report`, `test`, `debug`, `analysis`, `summary`, `reference`
4. **Description**: kebab-case, 1-5 words describing the session focus

**Examples**:
- `2025-10-14-phase-3.6-progress-day1-backend-hitl.md` - Daily progress log for backend HITL development
- `2025-10-14-phase-3.6-test-unit-hitl-nodes.md` - Unit test report for HITL nodes
- `2025-10-15-phase-3.6-analysis-progress-optimization.md` - Analysis of development progress vs plan
- `2025-10-20-phase-3.6-summary-complete.md` - Final completion summary for Phase 3.6

**Category Definitions**:
| Category | Purpose | Frequency |
|----------|---------|-----------|
| `plan` | Implementation plans | 1-2 per phase |
| `progress` | Daily/incremental progress logs | 1-3 per day |
| `report` | Completion reports | 1 per milestone |
| `test` | Test execution reports | As needed |
| `debug` | Debugging session logs | As needed |
| `analysis` | Strategic/retrospective analysis | 1-2 per week |
| `summary` | Phase/project summaries | 1 per phase |
| `reference` | Quick reference guides | 1-2 per phase |

**Why This Format?**
1. **Chronological Sorting**: Files naturally sort by creation date (`ls` or file browser)
2. **Phase Grouping**: Easy to filter files by `grep "phase-3.6"`
3. **Category Filtering**: Quick search for specific document types (`grep "test"`)
4. **Self-Documenting**: Filename alone provides context (date + phase + purpose)
5. **Git-Friendly**: Avoids special characters, safe for all operating systems
```

### C. 文件模板生成器

```bash
#!/bin/bash
# create_session_file.sh - Generate session file with template

PHASE="$1"      # e.g., phase-3.6
CATEGORY="$2"   # e.g., progress, test, report
DESC="$3"       # e.g., day5-frontend-integration

if [ -z "$PHASE" ] || [ -z "$CATEGORY" ] || [ -z "$DESC" ]; then
  echo "Usage: $0 <phase> <category> <description>"
  echo "Example: $0 phase-3.6 progress day5-frontend-integration"
  exit 1
fi

DATE=$(date +%Y-%m-%d)
FILENAME="${DATE}-${PHASE}-${CATEGORY}-${DESC}.md"

cat > "$FILENAME" << EOF
# Phase ${PHASE##*-} - ${DESC}

**Date**: $DATE  
**Category**: $CATEGORY  
**Status**: 🚧 IN PROGRESS

---

## 📋 Goals

- [ ] TODO: Define goals

---

## 🚀 Tasks

### Task 1: TODO

**Status**: 🚧 IN PROGRESS

---

## 📊 Results

TODO: Add results

---

## 📝 Next Steps

- [ ] TODO: Define next steps
EOF

echo "Created: $FILENAME"
```

使用示例：
```bash
./create_session_file.sh phase-3.6 progress day5-frontend-integration
# Output: Created: 2025-10-16-phase-3.6-progress-day5-frontend-integration.md
```
