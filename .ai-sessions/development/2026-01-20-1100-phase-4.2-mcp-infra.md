# Session: Phase 4.2 - MCP Infrastructure Design

**Date:** 2026-01-20
**Architect:** GitHub Copilot
**Phase:** 4.2 (MCP Infrastructure & Hexagonal Migration)

## Objective
Design the core "Contract" for the Model Context Protocol (MCP) Server integration. This involves:
1.  Defining the `McpTool` schema to standardize how tools are defined in the domain layer.
2.  Defining the `McpServer` protocol to abstract the underlying MCP server implementation (likely `mcp` python SDK or `fastmcp`).

## Design Decisions

### 1. Hexagonal Boundary
- **Domain Layer**: Defines `McpTool` (metadata) and `McpServer` (behavior). Independent of `mcp` SDK.
- **Infrastructure Layer**: Will use `mcp` (official SDK) or `fastmcp` to implement the server over Stdio/SSE.

### 2. Schema (`McpTool`)
We need a structure that holds:
- Name
- Description
- JSON Schema for arguments (Pydantic models can generate this)
- Execution handler (Callable)

### 3. Protocol (`McpServer`)
Needs methods to:
- `register_tool(tool: McpTool)`
- `serve()` (start the server loop)
- `list_tools()` (optional, for introspection)
