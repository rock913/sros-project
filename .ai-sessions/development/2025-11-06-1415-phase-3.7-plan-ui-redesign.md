## 📋 问题分析

### 当前痛点

1. **信息割裂**:
   - Manuscript 和 Asset Library 分离在两个独立的标签页
   - 用户需要切换标签页才能查看论文列表
   - 无法直观看到报告中引用了哪些论文

2. **会话管理不便**:
   - Session 列表只显示 ID 和时间
   - 缺少研究主题名称，难以快速定位
   - 没有会话状态和元数据的可视化

3. **缺少上下文关联**:
   - 查看报告时看不到相关论文
   - 查看论文时不知道是为哪个研究收集的
   - 缺少"从报告跳转到论文"的交互

---

## 🎯 设计目标

### 核心原则

1. **Context-First（上下文优先）**: 所有信息都在研究上下文中呈现
2. **Single Source of Truth（单一真相来源）**: Session 是所有数据的锚点
3. **Progressive Disclosure（渐进式披露）**: 默认显示关键信息，支持深入探索
4. **Inline References（内联引用）**: 报告中的引用可直接跳转到论文详情

## 📊 Session 列表设计细化

### Session Card 组件

```typescript
interface SessionCardProps {
  session: {
    id: string;
    researchTopic: string;
    status: 'completed' | 'in_progress' | 'paused' | 'failed';
    createdAt: Date;
    completedAt?: Date;
    paperCount: number;
    hasReport: boolean;
  };
  isSelected: boolean;
  onClick: () => void;
}
```

### 视觉设计

```
┌─────────────────────────────────────────┐
│ ✅ Transformer Architecture             │ ← 状态图标 + 主题（粗体）
│ 📅 2025-11-06 10:30                     │ ← 日期时间
│ 📊 15 papers | 📝 Report ready          │ ← 统计信息
│ ⏱️ Completed in 3m 45s                  │ ← 可选：执行时长
└─────────────────────────────────────────┘
   ↑ 选中状态：蓝色边框 + 浅蓝背景
```

### 状态图标映射

- ✅ `completed`: 绿色，研究完成且有报告
- 🔄 `in_progress`: 蓝色动画，正在执行
- ⏸️ `paused`: 黄色，等待 HITL 输入
- ❌ `failed`: 红色，执行失败
- 📝 `no_report`: 灰色，完成但未生成报告

### 排序和筛选

**默认排序**: 最新在前（`createdAt DESC`）

**筛选选项**:
- All Sessions
- Completed (✅)
- In Progress (🔄)
- With Reports (📝)
- Failed (❌)

**搜索**: 按研究主题名称搜索

## 🤔 待审核决策点

### 决策 1: 布局方案选择

**问题**: 方案 A（主-从布局）vs 方案 B（标签页增强）？

**推荐**: 方案 A
- ✅ 信息密度更高，减少上下文切换
- ✅ 更符合"深度融合"的需求
- ✅ 更适合 VS Code 的侧边栏布局习惯
- ⚠️ 需要更多的垂直滚动空间

**备选**: 方案 B
- ✅ 保留用户熟悉的标签页习惯
- ✅ 适合非常长的报告（减少滚动）
- ⚠️ 仍然存在一定的信息割裂

### 决策 2: Session 列表宽度

**问题**: 左侧 Session 列表应占多少宽度？

**选项**:
- A. 30% (推荐): 平衡显示，Session 标题可显示 20-25 个字符
- B. 25%: 给 Details 更多空间，但 Session 标题截断较多
- C. 可调节分割线: 用户可自定义，但增加复杂度

**推荐**: A (30% 固定)，后续可根据反馈考虑可调节

### 决策 3: 引用编号格式

**问题**: Manuscript 中的引用应该如何显示？

**选项**:
- A. `[1]` (推荐): 传统学术论文格式
- B. `(1)`: 圆括号格式
- C. `¹`: 上标格式
- D. `[Vaswani2017]`: 作者-年份格式

**推荐**: A `[1]`，最常见且易于点击

### 决策 4: Paper Card 默认状态

**问题**: Paper Card 的摘要应该默认展开还是折叠？

**选项**:
- A. 默认折叠 (推荐): 减少滚动，用户按需展开
- B. 默认展开: 信息更完整，但页面很长
- C. 智能展开: 前 3 篇展开，其余折叠

**推荐**: A (默认折叠)，保持页面简洁

### 决策 5: Session 列表分页

**问题**: Session 列表如何处理大量数据？

**选项**:
- A. 虚拟滚动 (推荐): 流畅体验，适合大量 Session
- B. 分页加载: 传统方式，简单但体验略差
- C. "Load More" 按钮: 简单实现，适合数据量不大的情况

**推荐**: C (初期) → A (优化)
- 初期实现 "Load More" (快速开发)
- 数据量增长后升级到虚拟滚动
