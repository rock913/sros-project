# Phase 3.7 Unified Research View - Testing Guide

## 🧪 Testing Instructions

### Prerequisites
1. Backend API must be running on `http://localhost:8000`
2. VS Code Extension Development Host ready

### Step 1: Launch Extension
```bash
# In VS Code
1. Open: vscode-extension folder
2. Press: F5 (Start Debugging)
3. Wait for: Extension Development Host window to open
```

### Step 2: Open Research Sessions View
```
1. Click: Activity Bar → Auto Researcher icon
2. Look for: "Research Sessions" panel (top panel)
3. Should see: Either session cards OR "No research sessions yet" message
```

### Step 3: Test Session List
**If you have sessions:**
- ✅ Each card should show:
  - Status icon (✅/🔄/⏸️/❌)
  - Research topic (truncated to 80 chars)
  - Paper count
  - Relative time (e.g., "5m ago", "3h ago")

**If empty:**
- ✅ Should show:
  - "No research sessions yet" message
  - "+ New Session" button

### Step 4: Test Session Selection
```
1. Click: Any session card
2. Expected: 
   - Card highlights with blue border
   - Right panel loads session details
   - Shows session info card
   - Shows manuscript (if available)
   - Shows papers list
```

### Step 5: Test Search
```
1. Type in: Search input box
2. Expected: Sessions filter in real-time
3. Clear search: Sessions reappear
```

### Step 6: Test Filters
```
1. Click: "Completed" chip
2. Expected: Only completed sessions visible
3. Click: "All" chip
4. Expected: All sessions visible again
```

### Step 7: Test Manuscript Features
**If manuscript exists:**
```
1. Look for: Research Manuscript section
2. Click: Reference link [1]
3. Expected: Scrolls to paper #1, highlights it for 3 seconds
4. Click: "Export MD" button
5. Expected: Save file dialog opens
6. Save: File should contain markdown content
```

### Step 8: Test Paper Features
```
1. Find: Any paper card
2. Click: "▼ Show Abstract" button
3. Expected: Abstract expands
4. Click: "▲ Hide Abstract" button
5. Expected: Abstract collapses
6. Click: "📄 Open PDF" (if available)
7. Expected: PDF opens in external browser
8. Click: "📋 Copy BibTeX"
9. Expected: 
   - Notification: "BibTeX citation copied to clipboard!"
   - Paste somewhere: Should see BibTeX citation
```

### Step 9: Test Refresh
```
1. Click: Refresh button (top-right of panel)
2. Expected: 
   - Sessions list reloads
   - Notification: "Research sessions refreshed!"
```

### Step 10: Test New Session
```
1. Click: "+ New Session" button
2. Expected:
   - Input dialog appears
   - Prompt: "Enter your research topic"
3. Enter topic: "Test Research Topic"
4. Click: OK
5. Expected: Research progress panel opens
```

## 🔍 Debugging Tips

### Check Backend
```bash
curl http://localhost:8000/health
# Should return: {"status": "ok"}
```

### Check Sessions API
```bash
curl "http://localhost:8000/analytics/sessions?limit=10"
# Should return: {"sessions": [...], "total": N}
```

### Check Console
```
1. In Extension Development Host
2. Press: Ctrl+Shift+I (Open DevTools)
3. Look for: Console errors
```

### Check Extension Logs
```
1. In main VS Code window
2. View → Output
3. Select: "Extension Host" from dropdown
4. Look for: "Failed to load sessions" or other errors
```
