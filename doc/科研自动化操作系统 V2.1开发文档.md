科研自动化操作系统 V2.1 (SROS) 开发规范书

1. 战略定位与愿景

SROS V2.1 旨在构建一个基于 VS Code 原生架构 与 Model Context Protocol (MCP) 的分布式智能科研环境。不同于传统的碎片化工具，SROS 将文献发现、逻辑推理与学术写作整合为一个连续的智能流，实现从“AI 辅助工具”向“科研操作系统”的范式转变。

核心目标
文献调研自动化：基于引文网络的深度检索与综述生成。

沉浸式写作：针对综述及长篇科普书籍的层级化撰写与优化。
引文闭环管理：与 Zotero 等文献管理软件实现原生双向同步。

2. SROS V3.0 架构规范 (核心架构层)

2.1 设计哲学：原生集成 (Native Integration)

不重新造轮子。利用 VS Code 的原生 UI 组件，确保用户在熟悉的环境中获得最低的认知摩擦。

2.2 三大核心交互组件

灵动导航大纲 (Primary Sidebar - TreeView)：

实现方式：利用 VS Code TreeView API。

功能：大纲不再只是标题，而是带有状态的“任务节点”（如：调研中、待扩写）。支持点击跳转、右键触发 AI 指令。

沉浸式编辑器 (Main Editor)：

实现方式：原生 VS Code 编辑器 + CodeLens + Decoration。

功能：引文悬浮预览、行间 AI 提示 (Ghost Text)、以及选中文本后出现的浮动极简操作栏（包含文献增强、科普转换、逻辑检查）。

科研助手面板 (Secondary Sidebar - Webview)：

实现方式：自定义 Webview 面板（遵循 VS Code 主题变量）。

功能：支持自然语言导航的意图输入框、引文决策卡片、以及 SOP 任务进度条。

3. 关键功能模块设计

3.1 引文引擎 (Citation Engine)

Zotero 深度集成：通过 zotero-mcp-server 实时校验本地数据库，生成兼容 Pandoc/Markdown 的 [@citekey]。

引文注入逻辑：AI 提出论点时弹出“证据卡片”，展示本地及 Arxiv 最匹配文献，由用户审批注入。

3.2 极简导航逻辑 (Long-Doc Navigation)

针对综述和书籍等长文本，系统实现“意图驱动”定位：

Command Palette：集成 Cmd+Shift+P，搜索“SROS: Go to section...” 实现全书快速跳转。

自然语言定位：用户在助手中输入“去结论部分”，编辑器自动通过 revealRange 定位视野。

3.3 技术路线：MCP 驱动的内核

内核层：基于 MCP 构建。

zotero-mcp-server: 处理本地引文元数据。

academic-search-mcp: 代理 Arxiv/PubMed 检索。

document-mcp: 负责长文档的分块加载与语义搜索 (RAG)。

4. 界面原型与交互设计 (UI/UX)

界面设计说明：左侧为灵动大纲节点，中间为支持引文高亮与 Ghost Text 的编辑器，右侧为极简的 AI 任务控制台。

4.1 模拟交互场景

场景：语义导航

用户输入："跳转到第二章关于鲁棒性的讨论"

系统响应：侧边大纲高亮 2.1 节，编辑器自动滚动至对应行。

场景：智能引文

用户选中文段并要求“寻找支持证据”。

系统响应：右侧面板弹出文献卡片，点击“确认”后在光标处插入 [@wang_2024] 并同步元数据。

5. 长期兼容性与扩展性规划

驱动化设计：任何新的数据源（如实验室数据库）只需编写 MCP 驱动程序即可挂载，无需更改 UI 代码。

标准兼容：原生支持 .md 和 .tex，底层采用 Pandoc 转换引擎，确保导出至 Word/LaTeX 的完美格式。

6. 开发路线图 (Roadmap)

阶段 1 (当前)：建立 VS Code 基础插件框架，实现 Zotero MCP 基本连接与大纲跳转。

阶段 2 (扩展)：完善多智能体协作流程，支持科普书籍的自动化语调转换。

阶段 3 (智能)：引入预测式预加载，基于写作上下文自动准备调研资料。
