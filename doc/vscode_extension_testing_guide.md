# VS Code 插件调试与手动测试指南

本文档提供了在本地开发环境中编译、运行和手动测试 "Auto-Researcher" VS Code 插件的详细步骤。

## 先决条件

- 你必须在 VS Code 中，并已通过 Dev Container 连接到**前端开发环境** (默认的 `devcontainer.json` 配置)。
- 你的终端已打开，并准备好执行命令。

---

## 测试步骤

### 步骤 1: 准备开发环境

在开始测试之前，请确保所有依赖都已正确安装。

1.  **导航到插件目录**:
    在 VS Code 的终端中，进入 `vscode-extension` 目录。

    ```bash
    cd /workspaces/gemini-fullstack-langgraph-quickstart/vscode-extension
    ```

2.  **安装/更新依赖**:
    如果你是第一次运行，或者 `package.json` 中的依赖有变动，请运行此命令。

    ```bash
    npm install
    ```

### 步骤 2: 编译插件代码

VS Code 插件使用 TypeScript 编写，需要先编译成 JavaScript 才能运行。

1.  **编译代码**:
    ```bash
    npm run compile
    ```
    此命令会读取 `tsconfig.json` 配置，并将 `src` 目录下的 `.ts` 文件编译到 `out` 目录中。

    > **提示**: 如果你希望在修改代码后自动重新编译，可以运行 `npm run watch`。

### 步骤 3: 启动插件进行测试

使用 VS Code 内置的调试功能来启动一个加载了你的插件的新窗口。

1.  **打开“运行和调试”视图**:
    点击 VS Code 左侧活动栏中的“运行和调试”图标 (▶️)。

2.  **启动调试会d话**:
    - 确认顶部的调试配置下拉菜单中选择了 **"Run Extension"**。
    - 按下 **`F5`** 键 (或点击旁边的绿色播放按钮)。

    此操作将打开一个**新的 VS Code 窗口**，这个窗口被称为“扩展开发主机”(Extension Development Host)。你的插件将自动在这个新窗口中被加载和激活。

### 步骤 4: 在“扩展开发主机”中手动测试

现在，你可以在这个新窗口中像最终用户一样与你的插件进行交互，验证其功能。

1.  **检查侧边栏**:
    - 在最左侧的活动栏中寻找你的插件图标 ("Auto Researcher")。
    - 点击图标，确认你的自定义侧边栏视图 (如 "Asset Library", "Manuscript") 是否正确显示。

2.  **执行命令**:
    - 打开命令面板 (`Ctrl+Shift+P` 或 `Cmd+Shift+P`)。
    - 输入你的命令名称，例如 `Show AI Control Panel`，然后按 Enter 键执行。
    - 观察插件的行为是否符合预期。

3.  **验证 UI 交互**:
    - 点击你在视图中添加的各种按钮 (例如，刷新按钮)。
    - 检查所有 UI 元素是否按预期工作。

---

## 开发迭代

当你修改了插件的源代码后，需要重复以下步骤来测试你的改动：

1.  **停止调试**: 关闭“扩展开发主机”窗口。
2.  **重新编译**: 在你的主开发窗口的终端中再次运行 `npm run compile`。
3.  **重新启动调试**: 按 `F5` 再次启动“扩展开发主机”窗口，然后验证你的改动。
